A Semigroup has an associative binary operation (combine).
A Monoid adds an identity element (empty).
A Group adds an inverse operation.

## Core.Instances

Instances for core algebraic types: Nat, Bool, List, Ordering.

Provides Eq, Ord, Monoid, Semiring, Semigroup, and Convertible
instances for the core data types.

**See also:** Core.Nat, Core.Bool, Core.List, Core.Ordering


- **`instance Eq(Nat)`**
  Eq(Nat): structural equality via recursive comparison.
  - `function (==) (x:Nat,y:Nat) : Bool`
- **`instance Ord(Nat)`**
  Ord(Nat): Z < Succ(anything), recursive comparison.
  - `function compare(x:Nat, y:Nat) : Ordering`
  - `function (<)(x:Nat, y:Nat) : Bool`
  - `function (>)(x:Nat, y:Nat) : Bool`
  - `function (<=)(x:Nat, y:Nat) : Bool`
  - `function (>=)(x:Nat, y:Nat) : Bool`
- **`instance Monoid(Nat)`**
  Monoid(Nat): combine = plus, empty = Z.
  - `function combine(x:Nat, y:Nat) : Nat`
  - `value empty`
- **`instance Semiring(Nat)`**
  Semiring(Nat): (+) = plus, (*) = mult, zero = Z, one = Succ(Z).
  - `function (+)(x:Nat, y:Nat) : Nat`
  - `function (*)(x:Nat, y:Nat) : Nat`
  - `value zero`

## HKT.Functor

Functor — types that can be mapped over.

A Functor provides fmap, which applies a function to every value
inside a container while preserving the container structure.

**See also:** HKT.Applicative, HKT.Monad


### Algebra: `Functor`

```tulam
algebra Functor(f:Type1)
```

Functor algebra for type constructors (Type1).


**Functions:**

- `function fmap(g: a -> b, x:f(a)) : f(b)`
  Apply a function to every value inside the functor.
  - **g**: The function to apply
  - **x**: The functor value
  - Returns: The functor with g applied to all contained values
  - Example: `fmap(Succ, Just(Z)) => Just(Succ(Z))`
**Laws:**

- *identity*: `law identity(x:f(a))`
- *composition*: `law composition(f:b -> c, g:a -> b, x:f(a))`

## HKT.Applicative

Applicative functor — functors with pure and apply.

Extends Functor with the ability to lift values into the functor (pure)
and apply wrapped functions to wrapped values (ap).

**See also:** HKT.Functor, HKT.Monad


### Algebra: `Applicative`

```tulam
algebra Applicative(f:Type1) extends Functor(f)
```

Applicative functor algebra.


**Functions:**

- `function pure(x:a) : f(a)`
  Lift a value into the functor.
  - **x**: The value to lift
  - Returns: The value wrapped in the functor
  - Example: `pure(1) : Maybe(Int) => Just(1)`
- `function ap(ff:f(a -> b), fa:f(a)) : f(b)`
  Apply a wrapped function to a wrapped value.
  - **ff**: Functor containing a function
  - **fa**: Functor containing a value
  - Returns: The result of applying the function inside the functor

## HKT.Monad

Monad — sequencing computations with context.

Extends Applicative with bind (>>=) for chaining computations where
each step can depend on the result of the previous one.

**See also:** HKT.Applicative, HKT.Functor


### Algebra: `Monad`

```tulam
algebra Monad(m:Type1) extends Applicative(m)
```

Monad algebra for sequential computation.


**Functions:**

- `function bind(x:m(a), f: a -> m(b)) : m(b)`
  Chain two computations — the second depends on the first's result.
  - **x**: The first computation
  - **f**: A function from the result of x to the next computation
  - Returns: The combined computation
  - Example: `bind(Just(1), \x -> Just(x + 1)) => Just(2)`
- `function seq(x:m(a), y:m(b)) : m(b)`
  Sequence two computations, discarding the first result.
  - **x**: The first computation (result discarded)
  - **y**: The second computation
  - Returns: The result of y
**Laws:**

- *left_identity*: `law left_identity(x:a, f: a -> m(b))`
- *right_identity*: `law right_identity(x:m(a))`
- *associativity*: `law associativity(x:m(a), f: a -> m(b), g: b -> m(c))`
